# Thesis Reader Loop

## Purpose

Read, screen, verify, discuss, and package research sources for user decision-making.

This loop is platform-independent. ChatGPT, Codex, Claude, Cursor, or local agents are runtimes.

## Product-Neutral Scope

Thesis Reader is for broad UX / HCI / AI PM research takeaway synthesis. It must not default to any specific product, device, company project, or user project as the frame for interpretation.

Do not anchor outputs to Robot Phone, phone hardware, embodied devices, or any other named product unless the source itself is about that product or the user explicitly requests that lens. Prefer generalizable interaction patterns, system capabilities, user-control issues, evaluation signals, and product-agnostic takeaway points.

## Flow

Source Intake
→ Triage Brief
→ Human Decision
→ Verified Reading
→ Discussion Memory
→ Paper Knowledge Package
→ Knowledge Base Handoff Gate

## Human Gates

Require user approval before:
- entering the full verified loop after triage;
- choosing controversial interpretations;
- turning research into strong product or portfolio recommendations;
- generating public-facing deck outputs;
- passing the Paper Knowledge Package into `research-knowledge-base`.

## Downstream Handoff

After verified reading and meaningful discussion, generate a Paper Knowledge Package and ask whether to enter the Research Knowledge Base Loop.
